Let's explain this picture.

⠀

It may seem complicated at first. But thanks to this, you will understand that it is not complicated at all.

Head Part

Next to the ascii art that says GDLG at the top, the numbers after the v indicate the version of the program. Below that is the welcome message, then it moves to the middle.

Middle

Name = Name of the created level (there are about 100 of them). Difficulty = Difficulty of the level (there are 4 easy and 6 difficult, except for the original difficulties) Song = Song ID of the level. You can copy the URL and check the song (you can choose from 1 to 1420812). Deco = Theme of the level, decoration idea (about 100, some of them may actually exist). D. Feature = The feature of the level that distinguishes it from other levels (about 100). Version = Version of the level (from 1.0 to 2.2). Duration = Duration of the level (from 30 seconds to 120 seconds). Objects = Number of objects created by the level (from 5100 to 40000)

Lower Part

R = Starts the program again in the same console. C = The program's production team. S = Closes the program completely. J = Creates a new level in JSON format. F = Runs the feedback html in the default browser. SS = Finds the location and size of the console and takes a screenshot appropriate for it. L = Opens all files ending with .log in the folder with notepad.

Lower Part (More)

T = This will open the tutorial html. There are other things that are not visible in the console to make it look fancy. They are: 1 = Opens the first music. 2 = Opens the second music. 3 = Opens the third music. NAME = Selects 5 random names. DIFF = Selects 5 random difficulties. SONG = Selects 5 random song IDs. DECO = Selects 5 random decoration ideas. FTR = Selects 5 random distinctive features. VER = Selects 5 random versions. DUR = Selects 5 random durations. OBJ = Selects 5 random object counts. GM = Selects 5 random 5 game modes. PORTAL = Selects 5 random 8 portals.
